Q2 — Injective authentication: User <-> GSS
===========================================
Query:

    query u : pid, n : nonce, sk : key;
      inj-event(GSSAcceptsUser(u,n,sk)) ==> inj-event(UserStarts(u,n)).

Meaning: whenever the GSS accepts a user session (after verifying the
operator verifier V_U, which binds the target UAV PID_j), a matching
UserStarts event must have occurred, on a one-to-one basis. This rules
out impersonation and replay of the operator on the User–GSS link.

Expected ProVerif result:

    RESULT inj-event(GSSAcceptsUser(u,n,sk)) ==> inj-event(UserStarts(u,n)) is true.
